## <span style={{display: "inline-flex", alignItems: "center", gap: "0.5em"}}><Icon name="spinner" spin /> Fast and feature-rich</span>

- <LinkRef to='meta-plugins'>Meta plugins</LinkRef> allow installing groups of plugins via a single, friendly label.
- [Packages](/ecosystem/category/-packages) offload the user from providing long and complex commands.
- [Annexes](ecosystem/category/-annexes) allow extending the plugin manager with new features.
- <LinkRef to='turbo-mode-zsh--53'>Turbo</LinkRef> mode yields **50-80%** faster Zsh startup.

## <span style={{display: "inline-flex", alignItems: "center", gap: "0.5em"}}><Icon name="heart" className="fa-beat" /> Neat and flexible</span>

- <LinkRef to='customizing-paths'>Customize paths</LinkRef>, use
  <LinkRef to='multiple-prompts'>multiple prompts</LinkRef> or create
  <LinkRef to='non-github-local-plugins'>your own</LinkRef> plugins.
- Supports <LinkRef to="oh-my-zsh-prezto">Oh My Zsh</LinkRef> and <LinkRef to="oh-my-zsh-prezto">Prezto</LinkRef> plugins and libraries. (<LinkRef to="help-migrate">migration</LinkRef>).
- Does not use <kbd>$FPATH</kbd>, loading multiple plugins doesn't clutter <kbd>$FPATH</kbd> with the same number of entries, e.g: 10, 15, or more.
- Code is immune to <kbd>KSH_ARRAYS</kbd> and other options typically causing compatibility problems.
- Do not require <kbd>root</kbd> access, and provide many workarounds e.g: setting so-called **shims** locally.

## <span style={{display: "inline-flex", alignItems: "center", gap: "0.5em"}}><Icon name="circle-info" className="fa-beat-fade" /> Familiarize and control</span>

- <LinkRef to='commands'>Visualize and manage</LinkRef> elements of the plugin: - **aliases**, **functions**,
  **bindkeys**, **zle widgets**, **completions**, **variables**.
- Quickly <LinkRef to="reports-and-statistics">familiarize</LinkRef> yourself with rich and easy-to-digest information.
- <LinkRef to='loading-and-unloading'>Load or unload</LinkRef> plugins, use the ability to
  <LinkRef to='completions-management'>manage</LinkRef> completions.
- <LinkRef to='configs-playground'>Docker playground</LinkRef>, test or propose configurations.
