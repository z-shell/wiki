
Available properties:

- `url`: The external URL of the SVG file (required)
- `alt`: Alt text for accessibility (required)
- `height`: Height of the SVG container (default: "auto")
- `width`: Width of the SVG container (default: "100%")
- `title`: Optional title for improved accessibility
- `href`: Optional URL to make the SVG clickable
- `className`: Optional additional CSS class
- `style`: Optional inline styles

### LinkRef

The `LinkRef` component allows you to reference common links from a centralized registry, making link management easier and more consistent across the documentation.


Links open in a new tab automatically if they are external:

You can customize the link text:


You can customize the link styling with different variants:


Available properties:

- `to`: Reference key to a predefined link in the registry (required)
- `children`: Text to display for the link (defaults to the key name)
- `newTab`: Whether the link should open in a new tab (defaults to true for external links)
- `variant`: Visual style of the link ("default", "button", or "subtle")
- `className`: Optional additional CSS class
- `title`: Optional title/tooltip for the link
- `id`: Optional ID for the link element

```jsx
<Player

  rows={24}
  autoPlay={false}
  loop={false}
  speed={1.5}
  idleTimeLimit={3}
  theme='monokai'
/>
```

### Usage Tips

- Use terminal recordings to demonstrate interactive features
- Keep recordings short and focused on specific tasks
- Use `speed` to adjust playback speed (1.0 is normal speed)
- Set `idleTimeLimit` to skip periods of inactivity
- Use `loop` for short, repeating demonstrations
- Add `controls` to allow users to pause/play the recording

For more information on using the Player component with asciinema recordings, see the [asciinema documentation](https://asciinema.org/docs/usage).

## Component Design Philosophy

Our custom components are designed with these principles in mind:

1. **Consistency**: Components provide a consistent visual language across documentation
2. **Clarity**: Each component has a clear purpose and enhances understanding
3. **Accessibility**: Components are designed to be accessible to all users
4. **Interactivity**: Where appropriate, components provide interactive features
5. **Flexibility**: Components can be combined and customized for different scenarios
6. **Simplicity**: Components are globally available without imports
7. **Type Safety**: All components have proper TypeScript definitions
8. **Maintainability**: Centralized component registration in MDXComponents.tsx

### Benefits for Documentation Authors

- **Reduced Boilerplate**: No need for import statements in MDX files
- **Discoverability**: All components are documented in one place
- **Consistent Styling**: Components match the site's design system
- **Better User Experience**: Enhanced interactivity and readability
- **Optimized Performance**: Components are optimized for production
- **Easier Contributions**: New contributors can focus on content, not implementation details

## Best Practices

When using these components in your documentation:

- Choose the appropriate component for the content you're presenting
- Don't overuse decorative components
- Maintain a consistent style throughout the document
- Test your documentation in both light and dark themes
- Consider internationalization implications
- **No imports needed** - all components are globally available

## Custom React Hooks

While not technically components, our custom React hooks are important parts of the codebase that can be used in your custom components. These hooks encapsulate reusable logic across components.

### useAsync

This hook simplifies handling asynchronous operations with loading states, error handling, and results.

```tsx
import {useAsync} from "@site/src/hooks";

// Define an async function
const fetchData = async (endpoint: string) => {
  const response = await fetch(`https://api.example.com/${endpoint}`);
  if (!response.ok) {
    throw new Error(`API error: ${response.status}`);
  }
  return response.json();
};

// Use the hook in your component
function DataComponent() {
  const {status, data, error, execute: fetchUserData} = useAsync(fetchData);

  useEffect(() => {
    fetchUserData("users/123");
  }, [fetchUserData]);

  if (status === "pending") {
    return <Spinner />;
  }

  if (status === "error") {
    return <div>Error: {error?.message}</div>;
  }

  if (status === "success") {
    return <div>Data: {JSON.stringify(data)}</div>;
  }

  return <button onClick={() => fetchUserData("users/123")}>Load Data</button>;
}
```

#### Hook Parameters

- `asyncFunction`: The async function to execute
- `immediate` (optional): Whether to execute immediately (default: `false`)
- `dependencies` (optional): Array of dependencies that trigger re-execution

#### Return Value

- `status`: Current status (`'idle'`, `'pending'`, `'success'`, or `'error'`)
- `data`: Result data (or `null` if not loaded)
- `error`: Error object (or `null` if no error)
- `execute`: Function to trigger the async operation
- `reset`: Function to reset to the idle state
