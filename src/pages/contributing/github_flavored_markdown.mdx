# GitHub Flavored Markdown

[GitHub Flavored Markdown](https://github.github.com/gfm/) (GFM) extends standard Markdown with additional features that make it more powerful for documentation and collaboration on GitHub. This guide covers both the standard Markdown syntax and the GitHub-specific extensions.

## Basic Syntax

GFM supports all the basic Markdown syntax, plus some GitHub-specific enhancements. You can either use the syntactic sugar that GFM provides or, since [Markdown can contain raw HTML](https://github.github.com/gfm/#raw-html), you can enter HTML tags manually for more control.

:::note
GitHub allows some HTML elements beyond what Markdown provides, but most raw HTML tags get stripped before rendering. Only tags that can be generated by GFM syntactic sugar, plus a few more, are whitelisted.
:::

## Alerts

GitHub supports special alert blocks for highlighting important information:

> [!NOTE]
> Highlights information that users should take into account, even when skimming.
> [!IMPORTANT]
> Crucial information necessary for users to succeed.
> [!WARNING]
> Critical content demanding immediate user attention due to potential risks.

The syntax uses blockquotes with a special identifier:

```markdown
> [!NOTE]
> This is a note.

> [!IMPORTANT]
> This is important information.

> [!WARNING]
> This is a warning.
```

## Disclosure Widgets with `<details>` and `<summary>`

A `<details>` element with a `<summary>` inside it can be used to [create a disclosure widget](https://developer.mozilla.org/en-US/docs/Web/HTML/Element/details). Add the `open` attribute to make it expanded by default:

<details open>
  <summary>This is the summary</summary>
  <p>And here are the details:</p>
  <ol>
    <li>Cash on hand: $500.00</li>
    <li>Current invoice: $75.30</li>
    <li>Due date: 5/6/19</li>
  </ol>
</details>

```html
<details open>
  <summary>This is the summary</summary>
  <p>And here are the details:</p>
  <ol>
    <li>Cash on hand: $500.00</li>
    <li>Current invoice: $75.30</li>
    <li>Due date: 5/6/19</li>
  </ol>
</details>
```

## Special Text Elements

### `<kbd>` and `<samp>`

Sample or quoted output from a computer program using `<samp>`: <samp>sample output</samp>.

User input with `<kbd>`: <kbd>help mycommand</kbd>.

For multiple keystrokes, nest the `<kbd>` tags:

<kbd>
  <kbd>Ctrl</kbd>+<kbd>c</kbd>
</kbd>

```html
<kbd><kbd>Ctrl</kbd>+<kbd>c</kbd></kbd>
```

For UI elements like buttons, you can nest `<samp>` inside `<kbd>`:

<kbd>
  <samp>OK</samp>
</kbd>

```html
<kbd><samp>OK</samp></kbd>
```

### `<sub>` and `<sup>`

Subscripts<sub>sub</sub> and superscripts<sup>sup</sup> with `<sub>` and `<sup>`:

```html
Subscripts<sub>sub</sub> and superscripts<sup>sup</sup>
```

### `<ins>` and `<del>`

Inserted text with `<ins>`: <ins>inserted</ins> and deleted text with `<del>`: <del>deleted</del>. These can both have the `cite` attribute.

```html
<ins>inserted</ins> and <del>deleted</del>
```

### `<var>`

Variables with `<var>`: <var>myVariable</var>.

```html
<var>myVariable</var>
```

### `<q>`

Inline quotes with `<q>`: <q>inline quote</q>. You can use the `cite` attribute on both `<q>` and `<blockquote>`.

```html
<q>inline quote</q>
```

## Definition Lists

Definition lists use `<dl>`, `<dt>`, and `<dd>` tags:

<dl>
  <dt>Publisher</dt>
  <dd>CNN and others</dd>
  <dt>Community admin</dt>
  <dd>Admins of groups</dd>
  <dt>Developer</dt>
</dl>

```html
  <dt>Publisher</dt>
  <dd>CNN and others</dd>
  <dt>Community admin</dt>
```

## Structured Data with `<div>`

The `<div>` tag is allowed with `itemscope` and `itemtype` attributes for [defining microdata](https://html5doctor.com/microdata/):

<div itemscope itemtype='https://schema.org/Movie'>
  <h6 itemprop='name'>Avatar</h6>
  <span>
    Director: <span itemprop='director'>James Cameron</span> (born August 16, 1954)
  </span>
  <span itemprop='genre'>Science fiction</span>
  <a href='https://youtu.be/0AY1XIkX7bY' itemprop='trailer'>
    Trailer
  </a>
</div>

```html
<div itemscope itemtype="https://schema.org/Movie">
  <h6 itemprop="name">Avatar</h6>
  <span>Director: <span itemprop="director">James Cameron</span></span>
  <span itemprop="genre">Science fiction</span>
  <a href="https://youtu.be/0AY1XIkX7bY" itemprop="trailer">Trailer</a>
</div>
```

## Tables

GFM supports tables with headers and alignment:

| Left-aligned | Center-aligned | Right-aligned |
| :----------- | :------------: | ------------: |
| Content      |    Content     |       Content |
| Text         |      Text      |          Text |

```markdown
| Left-aligned | Center-aligned | Right-aligned |
| :----------- | :------------: | ------------: |
| Content      |    Content     |       Content |
| Text         |      Text      |          Text |
```

## Task Lists

Create task lists with `- [ ]` for unchecked items and `- [x]` for checked items:

- [x] Completed task
- [ ] Incomplete task
- [ ] Another task

```markdown
- [x] Completed task
- [ ] Incomplete task
- [ ] Another task
```

## Footnotes

You can create footnotes using `[^1]` syntax[^1].

```markdown
You can create footnotes using `[^1]` syntax[^1].

[^1]: This is the footnote content.
```

[^1]: This is the footnote content.

## Automatic References

GitHub automatically creates references for:

- Commit SHAs: a5c3785ed8d6a35868bc169f07e40e889087fd2e
- Issue/PR numbers: #123
- @mentions: @username

## Code Blocks with Syntax Highlighting

GFM supports syntax highlighting in code blocks by specifying the language:

```javascript
  console.log("Hello, world!");
}
```

````markdown
```javascript

````

```mermaid
    B-->D;
    C-->D;
```
